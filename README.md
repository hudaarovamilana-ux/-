# 🤰 Telegram Bot for Pregnant Women

Telegram-бот для беременных с автоматической ежедневной рассылкой поддерживающих сообщений.

## 📋 Функционал

### Основные возможности:
- **Отслеживание беременности** по неделям с подробной информацией
- **Подсчет шевелений малыша** с историей за 7 дней
- **Информация об анализах** по триместрам
- **Профиль пользователя** с возможностью редактирования данных
- **Ежедневная рассылка сообщений** в 11:00 и 21:00 (по серверному времени)

### Ежедневные сообщения:
Бот отправляет 2 сообщения в день активным пользователям:
- **11:00** - утреннее сообщение
- **21:00** - вечернее сообщение

Сообщения выбираются случайно из 20 заранее заданных поддерживающих сообщений:
- 👶 От лица малыша (5 сообщений)
- 🤰 Поддержка (5 сообщений)
- 🥗 Напоминания (5 сообщений)
- 🧠 Осознанность / спокойствие (5 сообщений)

**Защита от дублей:** Сообщения не повторяются в течение одного дня для одного пользователя.

## 🚀 Установка и запуск

### 1. Клонирование репозитория

```bash
cd futurecity
```

### 2. Создание виртуального окружения

```bash
python -m venv venv

# Windows:
venv\Scripts\activate

# Linux/Mac:
source venv/bin/activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения

Скопируйте `.env.example` в `.env` и заполните его:

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```env
BOT_TOKEN=your_bot_token_from_BotFather
# Необязательно — путь к SQLite (Railway Volume и т.п.)
# DATABASE_PATH=/data/pregnancy_bot.db
```

**Как получить BOT_TOKEN:**
1. Найдите бота [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте полученный токен

### 5. Инициализация базы данных

База данных создается автоматически при первом запуске. Дополнительно инициализировать не нужно.

### 6. Запуск бота

```bash
python main.py
```

Бот запустится и начнет:
- Принимать сообщения от пользователей
- Отправлять ежедневные сообщения в 11:00 и 21:00
- Проверять новые недели беременности каждый час

## 🗄️ База данных

### Таблицы:

**users** - информация о пользователях:
- `user_id` - Telegram ID
- `week` - неделя беременности
- `due_date` - предполагаемая дата родов
- `last_period_date` - дата последней менструации
- `registered_date` - дата регистрации
- `username` - имя пользователя
- `height_cm` - рост
- `weight_kg` - вес
- `notifications_enabled` - включены ли уведомления (1/0)
- `last_notification_week` - последняя неделя уведомления
- `pregnancy_day` - день беременности
- `source` - источник данных (lmp/conception/due_date/manual)
- `date_input` - введенная дата

**daily_message_log** - лог отправленных ежедневных сообщений:
- `user_id` - ID пользователя
- `message_index` - индекс сообщения
- `send_time` - время отправки
- `schedule_type` - тип (morning/evening)
- `created_at` - время создания записи

**daily_sent_tracker** - защита от повторной отправки:
- `user_id` - ID пользователя
- `date` - дата
- `schedule_type` - тип (morning/evening)
- `message_sent` - отправлено ли сообщение (1/0)

**kick_counts** - подсчет шевелений:
- `user_id` - ID пользователя
- `date` - дата
- `count` - количество шевелений
- `start_time` - время начала
- `last_kick_time` - время последнего шевеления

**message_logs** - лог всех входящих сообщений:
- `user_id` - ID пользователя
- `username` - имя пользователя
- `full_name` - полное имя
- `chat_id` - ID чата
- `message_text` - текст сообщения
- `created_at` - время создания

## 📝 Добавление новых сообщений

Чтобы добавить новые сообщения в ежедневную рассылку, отредактируйте файл `messages.py`:

```python
DAILY_MESSAGES = [
    # Добавьте новые сообщения в конец списка
    "Ваше новое сообщение",
]
```

Сообщения будут автоматически включены в ротацию при следующей отправке.

## 🔧 Настройка времени рассылки

Чтобы изменить время ежедневной рассылки, отредактируйте файл `scheduler.py`:

```python
# Утреннее сообщение (измените hour и minute)
scheduler.add_job(
    send_daily_messages,
    trigger='cron',
    hour=11,  # Измените на нужное время
    minute=0,
    args=[bot, 'morning'],
    id='morning_messages',
    replace_existing=True
)

# Вечернее сообщение (измените hour и minute)
scheduler.add_job(
    send_daily_messages,
    trigger='cron',
    hour=21,  # Измените на нужное время
    minute=0,
    args=[bot, 'evening'],
    id='evening_messages',
    replace_existing=True
)
```

## 📊 Команды бота

- `/start` - начало работы / онбординг
- `/stats` - статистика пользователей (административная)

## 🛠️ Технологии

- **Python 3.8+**
- **aiogram 3.x** - современный асинхронный фреймворк для Telegram ботов
- **APScheduler 3.x** - планировщик задач для cron-like расписаний
- **SQLite** - встроенная база данных
- **Flask** - для health-check endpoint (при деплое на Railway)

## 🌐 Деплой на Railway

Проект готов к деплою на Railway. Убедитесь, что:
1. В переменных окружения Railway задан `BOT_TOKEN`
2. При необходимости задайте `DATABASE_PATH` для персистентного хранилища
